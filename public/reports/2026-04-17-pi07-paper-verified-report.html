01 / TLDR

三分鐘看懂 π0.7

突破不在「模型更強」，在「steerable」— 能被人用自然語言一步步 coach，把單句 5% 成功率推到 95%。發生在 $10k 量級的桌面雙臂上。

Physical Intelligence（PI）2026-04-16 發佈 π0.7，同日 TechCrunch 獨家採訪。核心是 compositional generalization：組合不同 context 學到的 skill 解訓練資料沒有的任務。氣炸鍋紅薯 demo — 訓練資料只見過「別的機器人推蓋子」和「放瓶子到爐灶」各一次，單句 prompt ~5%，分步 coach（按按鈕 → 開蓋 → 放紅薯 → 關蓋 → 設時間）→ ~95%。CONFIRMED · TechCrunch

護城河三層

架構不是護城河（Gemma 3 4B + 860M flow matching 都在 π0.6 paper 公開）。真正三層疊加：（1）硬件極簡 $3.5k/臂、$10–12k/站，humanoid 對手 $100k+；（2）Steerable 交互 把 user 變成線上資料回饋源；（3）任務選擇 只打視覺能贏的一半戰場，另一半讓給 humanoid-first。

Good intelligence compensates for bad hardware. — Sergey Levine · TechCrunch 2026-04-16 DIRECT QUOTE

PI 不是被迫用便宜硬件，是主動選擇，然後用模型與資料規模補硬件能力的空缺。賭注現在贏著 — 但邊界清晰。

02 / ANATOMY

模型解剖 — 5B 主體 + 14B 世界模型 + 多模態 Context

π0.7 不是 π0.6 加資料 — 在輸入端做了三處真架構擴充。

總參數 5B 主模型 + 14B 世界模型 + 400M 視覺編碼器。關鍵新增不在 action expert，在輸入端：BAGEL-based 世界模型產生視覺子目標圖，MEM 式編碼器壓縮視頻歷史。這兩件讓 steerability 從「prompt trick」升級為「訓練時學到的條件策略」。

相對 π0.6 的三件新增

  • 14B BAGEL world model（mixture-of-transformers）：生成多視角 subgoal 圖像，推理時 condition 進 VLM。最大新增件。Paper
  • 400M MEM 視覺編碼器：壓縮 ≤6 幀/相機歷史（stride 1 s）為 token 序列，給模型短時記憶。Paper
  • 多模態 context conditioning with dropout：訓練同時灌 language + metadata（speed/quality/strategy 數值）+ control modality + subgoal image，推理任一組件可關 — zero-shot 與 coaching 是同一個 conditional policy 的兩個切片。Paper

保留自 π0.6

  • KI + FAST tokens：VLM/action expert 梯度隔離 + FAST autoregressive tokenizer co-training。
  • Flow matching head：860M transformer，fixed 50-step action chunk（50 token，非 50 Hz；控制頻率 paper 未揭）。Paper
  • RECAP 蒸餾：π0.6 離線 RL post-training（獨立 670M value VLM + offline advantage conditioning），π0.7 以 strategy metadata 把 specialist 行為蒸餾回通用模型。Paper

這件事的意義

Steerability 是訓練目標，不是 prompt trick。每個 context 組件訓練時可被 dropout，模型學到「給 subgoal 圖就跟；給 metadata 就按那速度；都沒給就憑 task prompt 試」— 一個 conditional policy distribution。把「能被 coach」當 ML 目標來訓，這是 π0.7 的核心 insight。

03 / STEERABLE

Steerable — π0.7 的真正突破

相對 π0.6 的唯一明確差異：模型學會「被 coach」。

Compositional generalization 老詞談了十年。π0.7 的差異在於不要求模型自主組合（仍不穩），而是運行時讓人類語言引導做組合。哲學轉向：從「one-shot 自主」→「human-in-the-loop，user 即線上數據源」。

氣炸鍋紅薯 demo

  • 訓練資料「氣炸鍋」僅 2 樣本：別的機器人推蓋子一次 + 開源資料放瓶子到爐灶一次。
  • 單句 prompt 成功率 ~5%；分步 coach（按鈕 → 開蓋 → 放紅薯 → 關蓋 → 設時間）→ ~95%。TechCrunch
  • 這不是 prompt engineering，是 steerability — 模型能聽懂中途糾正。
為何 steerable > benchmark 漲分

User 成 on-the-fly 數據源（「這步錯了，這樣做」）→ 部署即 free online data collection，每個 B2B pilot 客戶都在免費標 failure case。Google / Figure / Tesla 目前無此機制。

部署極限 demo（TechCrunch）

  • 連續咖啡 13 小時不中斷
  • 新家庭疊衣 2 小時不中斷
  • 工廠真實環境包裝裝箱

關鍵不是「能做」，是「能連續做」。VLA 常見失敗是 drift — 誤差累積幾十分鐘後任務崩。13 小時不崩 = 訓練成熟度訊號。JUDGMENT

Steerable 的邊界

複雜多步任務（「做個吐司」）仍無法單句完成。π0.7 仍非「通用自主代理」，是「懂得怎麼 coach」的助手。離 Figure/1X 行銷的「humanoid 幫你做所有事」尚遠。

04 / HARDWARE

硬件深潛 — $3,500 的臂憑什麼

整個 π0.7 故事的基石：硬件是 humanoid-first 對手的 1/10；資料規模、flywheel、B2B 部署速度都建立於此。

Levine 親口：單臂 vendor ~$3,500，材料 <$1,000。產品圖鉻牌 π ARM-1017、π ARM-1091 — 命名慣例「π ARM」= Physical Intelligence Arm，OEM 不會允許客戶覆蓋鉻牌，PI 自研 + 已序列化量產、fleet ≥ 數十台 INFERRED。AgileX PiPER 同 class（6-DoF 球腕、1.5kg payload、626mm reach、$2,499 零售）— 證明 BOM 是商品化項目，但非 OEM 來源。

「自研 + $3,500 vendor 價」如何解

最合理解釋：PI 設計 + 合約製造。PI 持設計 IP 與命名權，代工廠報價 $3,500（物料 <$1k + 加工 + margin）。同構於 Apple/富士康、Tesla/Panasonic 電芯。JUDGMENT

平台構型

  • 雙臂 tabletop（ALOHA 風格）、固定基座（π0.5 曾 mobile，π0.6 轉回固定），工作空間 0.4–0.6m 高桌面。

腕部大黑圓柱

並非觸覺感測器，是獨立外掛 roll 軸關節模組（電機 + 減速器 + 編碼器 + 驅動器 + 制動器一體化）。尺寸大因三個連動原因：

  • 低減速比 ~10:1 — 刻意 backdrivable，operator 可直接推動從臂採 demonstration。
  • 模組化共用 SKU — 跨平台同款 actuator 降 BOM 複雜度。
  • 連續 duty cycle 散熱 — 13 小時咖啡不停要 actuator 不過熱。

末端執行器細節

  • 夾爪側 AprilTag + 可讀字符 ID：設備標識/標定，不排除跨臂 pose 閉環。
  • 刀柄纏綠色防滑套 — 給物加摩擦而不給夾爪加觸覺：用 prop hack 延伸「good intelligence compensates for bad hardware」原則。

05 / ABSENCE

觀測空間 — 缺席的一切，是策略

PI 缺的不只是感測器 — 還有仿真、深度幾何、主動控制。沒一樣是巧合。

其他 VLA 團隊標配的三大類 PI 全部不用：多模態感測（觸覺/力矩/深度）、仿真訓練、主動力控。這是路線設計，不是「做不到」。把這三樣從成本結構拿掉 = 10× 硬件便宜的來源，代價是模型能力必須補上缺口。

每個「缺席」都有對應「替代」。PI 不是「漏做」，是把傳統機器人學用昂貴感測器/仿真/MPC 解的問題，換成用模型 × 資料規模解。2023 年 LLM 打敗 rule-based NLP 的同一套打法搬到機器人。

賭贏類別

06 / MOATS

護城河 — 假的與真的

不是一條，是五層疊加。分界線：別人光看 paper 能不能抄？

假護城河

#1
模型架構

Gemma 3、flow matching、KI、Recap 全公開，任何 research team 可復現。

#2
人才密度

Sergey / Chelsea / Karol 強，但 Google / Tesla / Figure 團隊絲毫不遜色。

#3
硬件 IP

自研沒錯，但走商品化 BOM 路線（6-DoF 球腕、並指夾爪、商用相機），有機電團隊半年可複製同型。自研 ≠ 護城河。

#4
消費者品牌

投資圈外無認知。B2B 客戶關係才 12 個月，可替代。

真護城河（五層）

1 · 硬件成本結構

$10–12k × 100 站 = $1.1M fleet；humanoid 對手同規模需 $10M。這不只是融資差，是資料飛輪轉速差 — 一年下來 PI 多一個量級有效訓練資料。

2 · Data Factory 生產函數

$50–100/hr loaded cost 產高品質示範 = operator × task design × prop mgmt × hardware fleet × QA × labeling 的乘積。任一環節想抄都做不到。JUDGMENT

3 · Steerable = 免費 online data 機制

π0.7 獨有。B2B user 每次糾正 = 免費標 failure case。Google / Tesla 目前無。JUDGMENT

4 · Cross-embodiment 經驗

已在 AgileX-class / UR-class / Fetch / Stretch / Aloha 多平台訓練。每多一平台泛化邊際成本下降。

5 · Operations tacit knowledge

demo 指令怎麼寫、operator 怎麼訓、什麼失敗標 mistake=1、哪些回收重錄 — 全沒寫在論文。Palantir 式 Forward-Deployed 運營護城河。

算法是公開的，硬件是便宜的，資料與運營是隱形的。PI 把護城河建在最便宜也最難抄的那層。 — 本報告核心框架 JUDGMENT
07 / THREATS

誰能追上 PI？

分戰場問題。humanoid-first 與 PI 不打同類任務；真正對手是其他 vision-first VLA。

媒體把「Figure / 1X / Tesla vs PI」框成同場競賽是錯的。Humanoid 押注「人形 + 主動力控 + 多模態感測」那一半任務；PI 押注「桌面/固定基座 + 純視覺」另一半。真正對標者是 Google DeepMind 與智元。

Bear case — PI 會輸的四種劇本

  1. Sim-to-real 突破：NVIDIA GR00T / CMU 在 2026–27 讓仿真 ≈ 真實，PI 的 teleop 投入瞬間折舊。
  2. 觸覺 unlock 新類別：倒咖啡、插插頭、折毛巾 purely visual 卡 70%，觸覺能推 95%+，「硬件極簡」變主要技術債。
  3. Google open weight：Gemini Robotics 某版開權重，B2B 客戶先試免費的。
  4. Humanoid 突破純視覺極限：Figure / Tesla 以主動力控 + 觸覺把 humanoid 家務做到 80%+，「桌面雙臂」形態受限。
